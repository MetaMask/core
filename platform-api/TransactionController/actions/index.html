TransactionController:failTransaction​

Source: packages/transaction-controller/src/TransactionController-method-action-types.ts:151

Mark a transaction as failed, transitioning it through the standard failure path.

Unlike updateTransaction, this emits the transaction lifecycle events (transactionFailed, transactionStatusUpdated, transactionFinished), so downstream subscribers such as the bridge status controller and metrics are notified. Intended for callers that finalize a transaction out-of-band, for example the smart transactions controller when the relay cancels a smart transaction that never landed on chain.

Parameters:

NameDescription
transactionIdThe ID of the transaction to mark as failed.
errorThe error describing why the transaction failed.

Handler signature:

(transactionId: string, error: Error) => void
